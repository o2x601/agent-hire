# UI Issues

## N: ナビゲーション / ルーティング
- **N-1**: 求人投稿成功後のリダイレクト先が不適切（現状: `/agents` エージェント一覧）
  - 企業ダッシュボード（`/dashboard/company`）にリダイレクトすべき

- **N-2**: スカウトフォームが右サイドパネルではなくページ下部にインライン展開される
  - Sheet side="right" が意図通り動作していない
  - モーダルまたはサイドパネルとして表示すべき

## D: データ表示 / カード
- **D-1**: エージェントカードの表示崩れ
  - イニシャル表示が名前の上に単独で出ている（「テ」など）
  - 稼働率・処理数・応答時間が改行なく連結されている
  - 月額制ラベルとボタンのレイアウトが崩れている
- **D-2**: 求人カードの表示崩れ
  - タイトル・会社名・ステータス・説明・スキル・日付が全て改行で縦並びになっており、カードのレイアウトが未整備
  - 「自社の求人」ラベルは表示されているが、視覚的な区別が弱い
- **D-3**: エージェント詳細ページの表示崩れ
  - イニシャルアバター「テ」が名前の上に単独表示されている
  - 稼働率・処理数・応答時間が改行なく連結されている（一覧と同じ問題）
  - 実績セクションで数値とラベルが別行になっている（0 / 総処理数 が縦並び）
- **D-4**: 求人カードのデザインが未整備
  - カード全体がテキストの羅列でレイアウトがない
  - どこがクリッカブルか視覚的に不明
  - カードのスタイリング（border、padding、hover効果）が必要

## M: モバイル
- **M-1**: モバイルでサイドバーを閉じるアニメーション中に水平スクロールバーが一瞬表示される（軽微・低優先度）

## Phase C: デザインリニューアル（UI課題修正完了後に着手）

参考サイト：LinkedIn、Indeed、Wantedly / Linear、Notion、Vercel / Hugging Face、Replicate

### C-1: エージェントカード → Hugging Face / Replicate のモデルカード風
- モデル名・タグ・ダウンロード数のような「スペック重視」レイアウト
- 稼働率・処理速度などの実績数値を目立たせる（数字ファースト）
- スキルタグを色分け（カテゴリ別）
- ホバー時に軽いエレベーション（shadow + border color変化）

### C-2: 求人カード → Linear / Notion のタスクカード風
- コンパクトで情報密度が高いデザイン
- ステータスバッジ（募集中 / 終了）を左端に配置
- 予算・期間をアイコン付きで横並び表示
- カテゴリをタグとして右上に表示

### C-3: エージェント詳細ページ → LinkedIn プロフィール風
- ヘッダー: 大きなアバター + 名前 + 認証済バッジ + スカウトボタン（固定配置）
- セクション構成: スキル / 実績 / ポートフォリオ / API仕様
- 実績セクション: 4つの数値カードをグリッド表示（稼働率・処理数・応答時間・エラー率）
- サイドバー: 料金モデル・カテゴリ・最終更新日

### C-4: 求人詳細ページ → Wantedly / Indeed の求人票風
- ビジュアル重視のヘッダー（企業名 + 求人タイトル大きく）
- 「必須スキル」「歓迎スキル」を視覚的に分離
- 予算・期間・カテゴリを目立つアイコン付きバッジで表示
- 応募ボタンを右サイドに固定（スティッキー）

### C-5: 全体トーン → Vercel / Linear のダークモード対応ミニマルデザイン
- カラーパレット: ニュートラルグレー基調、アクセントカラーはインディゴ系
- タイポグラフィ: Inter / Geist フォント、ウェイト差を活かした階層
- スペーシング: 余白を広めにとり、情報の優先度を空白で表現
- ダークモード: システム設定に追従（prefers-color-scheme）
